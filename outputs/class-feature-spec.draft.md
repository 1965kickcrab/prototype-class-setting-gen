# 클래스 기능 명세 (초안)

## Spec Panel 1: 예약 신청 - 클래스 선택

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web (모바일) |
| 유저 | 일반 (보호자) |
| 화면명 | 예약 신청 |
| 화면 경로 | 예약 > 예약 신청 |
| Case | 반려견을 선택한 뒤 클래스를 선택하는 경우 |

### Spec Description 1: 클래스 선택 영역

| Item | Content |
|---|---|
| Description Type | Button |
| Area | 클래스 선택 |

- 반려견을 선택하기 전에는 선택 영역을 비활성화하고 `반려견을 먼저 선택해주세요`를 표시한다.
- 선택한 반려견 모두가 배정받은 클래스만 선택 목록에 노출한다.
- 선택 가능한 공통 클래스가 없으면 선택 영역을 비활성화하고 `클래스를 선택해주세요`를 표시한다.
- 클래스를 선택하면 선택 영역에 클래스명을 표시한다.

### Spec Description 2: 클래스 선택 바텀시트

| Item | Content |
|---|---|
| Description Type | Option |
| Area | 클래스 목록 |

- 클래스 선택 영역을 누르면 `클래스 선택` 바텀시트를 연다.
- 클래스는 단일 선택한다.
- 현재 선택된 클래스는 선택 상태로 표시한다.
- 바텀시트 외부를 누르면 선택을 변경하지 않고 닫는다.

### Spec Description 3: 반려견 또는 클래스 변경 시 예약일

| Item | Content |
|---|---|
| Description Type | State |
| Area | 선택한 예약일 |

- 반려견 구성을 변경하면 기존에 선택한 예약일을 모두 해제한다.
- 클래스를 변경하면 기존에 선택한 예약일을 모두 해제한다.

## Spec Panel 2: 예약 신청 - 날짜 가능 여부

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web (모바일) |
| 유저 | 일반 (보호자) |
| 화면명 | 예약 신청 |
| 화면 경로 | 예약 > 예약 신청 |
| Case | 클래스 선택 후 예약일을 선택하는 경우 |

### Spec Description 1: 날짜 선택 가능 상태

| Item | Content |
|---|---|
| Description Type | State |
| Area | 예약 달력 |

- 선택한 클래스의 운영 요일이 아닌 날짜는 선택할 수 없다.
- 선택한 반려견 중 한 마리라도 같은 날짜에 취소되지 않은 예약이 있으면 해당 날짜는 선택할 수 없다.
- 해당 날짜의 클래스 예약 인원과 새로 선택한 반려견 수의 합이 정원을 초과하면 `정원마감`으로 표시하고 선택할 수 없다.
- 과거 날짜는 선택할 수 없다.

| 항목 | 내용 |

### Spec Description 1: 예약 데이터의 클래스 정보

| Item | Content |
|---|---|
| Description Type | Value |
| Area | 생성된 예약 |

- 예약 건마다 선택한 클래스의 식별자와 클래스명을 저장한다.
- 예약 생성 시점의 클래스 식별자·이름·정원을 스냅샷으로 함께 저장한다.

## Spec Panel 4: 예약 상세 - 예약 정보

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web (모바일) |
| 유저 | 일반 (보호자) |
| 화면명 | 예약 상세 |
| 화면 경로 | 예약 > 예약 상세 |
| Case | 예약 정보를 확인하는 경우 |

### Spec Description 1: 클래스 항목

| Item | Content |
|---|---|
| Description Type | Value |
| Area | 예약 정보 섹션 |

- `예약 정보` 섹션에 `클래스` 항목을 추가한다.
- 예약에 저장된 클래스명을 표시한다.
- 클래스명이 없는 기존 예약은 `클래스 정보 없음`을 표시한다.

## Policy Panel

### 예약 상세의 클래스 표시 단위

- affected scope: 예약 상세 > 예약 정보 > 클래스
- confirmed policy: 예약 이력 상세은 예약 그룹의 첫 번째 예약에 저장된 클래스명을 표시한다.
- needs confirmation: 일자별 예약 상세에는 여러 클래스의 예약이 함께 포함될 수 있다. `예약 정보`에 단일 클래스명을 표시할지, 반려견별 카드에만 표시할지 결정이 필요하다.
